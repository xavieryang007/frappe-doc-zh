# 16.5 列出文档

## 列出文档 (Listing Documents)

要列出文档，请向 `/api/resource/{doctype}` 发出 GET 请求。

**示例请求：**
```
GET /api/resource/Person
```

> 所有列表都按 20 个项目分页返回。要更改页面大小，可以传递查询参数 `limit_page_length`。要请求后续页面，传递 `limit_start`。

响应以 JSON 对象形式返回，列表在键 `data` 中作为数组。

**响应：**
```json
{
  "data": [
    {
      "name": "000000012"
    },
    {
      "name": "000000008"
    }
  ]
}
```

默认情况下，列表中只包含 `name` 字段。要添加更多字段，你可以在 GET 请求中传递 `fields` 参数。`fields` 必须是一个包含字段名的 JSON 数组。

**带字段的示例请求：**
```
GET /api/resource/Person/?fields=["name","first_name"]
```

**响应：**
```json
{
  "data": [
    {
      "first_name": "Jane",
      "name": "000000012"
    },
    {
      "first_name": "John",
      "name": "000000008"
    }
  ]
}
```

你可以通过传递查询参数 `filters` 使用 SQL 条件过滤列表。`filters` 必须是一个包含一个或多个过滤器的 JSON 数组。每个条件是格式为 `[{doctype}, {field}, {operator}, {operand}]` 的数组。

**示例：获取所有名字为 "Jane" 的人的姓名（ID）：**
```
GET /api/resource/Person?filters=[["Person","first_name","=","Jane"]]
```

**响应：**
```json
{
  "data": [
    {
      "name": "000000012"
    }
  ]
}
```

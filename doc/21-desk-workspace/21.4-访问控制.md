# 21.4 访问控制

工作区（Workspace）的访问控制涉及模块权限和角色设置，通过双重机制来管理用户对工作区的访问权限。

## 1. 模块（Modules）控制

- 用户对不同模块的访问权限决定了他们能否看到相应的标准工作区。
- **示例**：如果用户"John Doe"没有"网站（Website）"模块的访问权限，则无法看到"Website Workspace"。
- 标准工作区的可见性直接与用户被授权的模块关联。

## 2. 角色（Roles）控制

- 即使用户拥有某模块的访问权限，仍可通过角色进一步限制其查看工作区。
- **示例**：
  - 用户"Jack Doe"是经理，拥有"Workspace Manager"角色。
  - 他希望只有具备"Website Manager"角色的用户才能访问"Website Workspace"。
  - 若用户"Jane Doe"有模块权限但无相应角色，则无法看到该工作区。

## 3. 默认工作区（Default Workspace）设置

- 用户首次登录时显示的工作区可以自定义。
- 默认情况下，用户会看到上次访问的工作区。
- 可通过用户设置中的"Default Workspace"字段（位于"My Settings"中）进行配置。

## 管理方法总结

### 权限配置
- 通过"用户权限"设置模块访问和角色分配。

### 工作区可见性
- 结合模块和角色双重控制，实现灵活权限管理。

### 默认设置
- 为每个用户指定个性化的工作区入口。

## 操作指南

### 设置模块权限
1. 进入用户管理界面
2. 选择目标用户
3. 在权限设置中勾选相应模块
4. 保存设置

### 设置角色权限
1. 进入工作区管理界面
2. 选择目标工作区
3. 配置访问角色要求
4. 保存设置

### 设置默认工作区
1. 进入用户个人设置
2. 在"My Settings"中找到"Default Workspace"字段
3. 选择希望设为默认的工作区
4. 保存设置

## 注意事项

- 模块权限是基础，角色权限是进一步的精细化控制
- 用户必须同时满足模块和角色要求才能访问工作区
- 默认工作区设置仅影响用户首次登录时的显示，不影响实际权限

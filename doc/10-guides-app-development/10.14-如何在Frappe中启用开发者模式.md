# 10.14 如何在Frappe中启用开发者模式

## 主要功能说明

启用开发者模式后，您将获得以下关键功能：

1.  **源代码集成**：在 Frappe 桌面界面中，您对 DocType、打印格式、工作流、网页表单等**应用元数据**所做的任何新建或修改，都会被直接写入到对应应用的源代码目录中（如 `apps/your_app/your_app`），而不是仅保存在数据库中。这使得这些更改可以通过版本控制系统（如 Git）进行管理。
2.  **访问开发工具**：界面会提供更多面向开发者的选项和快捷方式，例如：
    - "转到源文件"链接，方便快速跳转。
    - 更便捷地创建和应用补丁。
3.  **调试与开发**：这是进行应用定制和开发的必要环境，确保您的开发工作与代码库同步。

## 启用方法

### 方法一：使用 Bench 命令（推荐）

这是最简便和标准的方法。

1.  打开终端，并切换至您的 Frappe 基准目录（`frappe-bench`）。
2.  运行以下命令，为您的特定站点启用全局开发者模式：

```bash
bench --site [您的站点名称] set-config developer_mode 1
```

例如，如果您的站点名为 `mysite.local`，则命令为：

```bash
bench --site mysite.local set-config developer_mode 1
```

*注意：较新版本的 Bench 可能使用 `-g` 标志来设置全局配置。如果上述命令无效，可以尝试：*

```bash
bench --site [您的站点名称] set-config -g developer_mode 1
```

### 方法二：手动编辑配置文件

1.  找到您的站点配置文件。路径通常为：
    `frappe-bench/sites/[您的站点名称]/site_config.json`
2.  使用文本编辑器打开此文件。
3.  在文件的 JSON 对象内，添加或确保存在以下一行：

```json
"developer_mode": 1
```

完整的文件内容可能类似这样：

```json
{
  "db_name": "your_db_name",
  "db_password": "your_password",
  "developer_mode": 1,
  ...
}
```

4.  保存文件。

## 启用后的必要步骤

无论使用哪种方法启用开发者模式，**完成后都必须清除缓存**，以使更改生效。

在基准目录下运行以下命令：

```bash
bench --site [您的站点名称] clear-cache
```

## 可选步骤：安装开发依赖

某些应用可能需要额外的 Python 包来支持开发模式下的构建和测试。您可以使用以下命令安装这些开发依赖：

```bash
bench setup requirements --dev
```

## 总结

要启用 Frappe 的开发者模式：

1.  **核心操作**：使用 `bench set-config` 命令或手动编辑 `site_config.json` 文件，设置 `"developer_mode": 1`。
2.  **关键步骤**：启用后务必执行 `bench clear-cache` 来清除缓存。
3.  **功能目的**：此模式将您的界面操作与应用程序的源代码库连接起来，是进行可持续应用开发的基石。

完成以上步骤后，刷新 Frappe 桌面，您就会发现开发者模式已激活，并可以开始进行与源代码同步的开发工作了。

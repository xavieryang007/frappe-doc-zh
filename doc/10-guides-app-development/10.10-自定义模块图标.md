# 10.10 自定义模块图标

## 创建 SVG 图标文件

- 在您的应用目录中创建一个 SVG 格式的图标文件。
- 推荐存放路径：`[你的应用名]/assets/[应用名]/images/图标名.svg`
  例如：`assets/my_app/images/custom_icon.svg`

## 配置模块图标路径

- 编辑应用中的 `desktop/config.py` 文件。
- 在 `get_data()` 函数中，为模块指定 `icon` 属性，值为 SVG 文件的相对路径。

### 示例代码（`desktop/config.py`）：

```python
from frappe import _

def get_data():
    return {
        "你的模块名": {
            "color": "orange",       # 模块颜色（可选）
            "icon": "assets/my_app/images/custom_icon.svg",  # 图标路径
            "label": _("你的模块显示名称"),  # 模块标签（支持国际化）
            "type": "module"          # 类型为模块
        }
    }
```

## 图标加载机制

- 图标首次访问时会通过 AJAX 加载，之后会被缓存并直接渲染。
- 确保 SVG 文件路径正确，且应用静态资源已正确打包（可通过 `bench build` 更新资源）。

---

## 样式与设计建议

### 图标格式要求

- **必须使用 SVG 格式**，确保矢量缩放不失真。
- 避免使用过复杂的矢量路径，以保证加载性能。

### 颜色与尺寸

- 图标会继承模块配置中的 `color` 属性（如上例中的 `"orange"`），但具体显示效果取决于前端主题。
- 推荐使用单色或扁平化风格的 SVG，以便更好地适应主题色。

### 推荐资源

- 如需获取高质量、低成本的 SVG 图标，推荐使用 **[The Noun Project](https://thenounproject.com/)**（注：部分图标需授权）。

---

## 验证步骤

1. 完成配置后，重启 Frappe 服务：`bench restart`。
2. 刷新桌面页面，检查模块图标是否正常显示。
3. 如未显示，检查浏览器控制台是否有 404 错误（图标路径是否正确）。

---

## 扩展说明

- 此方法适用于 Frappe 框架下的所有自定义模块。
- 如需为模块设置多个图标或动态图标，需通过前端脚本进一步扩展（超出当前指南范围）。

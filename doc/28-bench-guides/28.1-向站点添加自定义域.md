# 28.1 向站点添加自定义域

## 概述
您可以为站点添加多个自定义域。只需运行以下命令：

```bash
bench setup add-domain [所需域名]
```

运行命令后，系统将提示您选择要为哪个站点设置自定义域。

## 配置SSL选项
您还可以使用以下选项为自定义域设置SSL：

- `--ssl-certificate` [证书路径]
- `--ssl-certificate-key` [证书密钥路径]

### 示例
```bash
bench setup add-domain custom.erpnext.com --ssl-certificate /etc/letsencrypt/live/erpnext.cert --ssl-certificate-key /etc/letsencrypt/live/erpnext.key
```

## 域配置存储位置

域配置存储在相应站点的 `site_config.json` 文件中，格式如下：

```json
"domains": [
  {
    "ssl_certificate": "/etc/letsencrypt/live/erpnext.cert",
    "domain": "erpnext.com",
    "ssl_certificate_key": "/etc/letsencrypt/live/erpnext.key"
  }
],
```

## 生效配置

为了使自定义域生效，您需要重新生成Nginx配置并重新加载Nginx服务：

1. 运行以下命令重新生成Nginx配置：
   ```bash
   bench setup nginx
   ```

2. 重新加载Nginx服务：
   ```bash
   sudo service nginx reload
   ```

## 总结步骤

1. 使用 `bench setup add-domain` 添加自定义域
2. 可选：通过 `--ssl-certificate` 和 `--ssl-certificate-key` 配置SSL
3. 检查 `site_config.json` 确认域配置
4. 重新生成并重新加载Nginx配置使更改生效

如果您需要进一步帮助，请参考Frappe官方文档或相关社区资源。

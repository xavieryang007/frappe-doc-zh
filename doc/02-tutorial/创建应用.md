# 创建应用 (Create an App)

## 创建Frappe应用的步骤

### 前提条件

- 确保您位于Bench目录中（可通过`bench find .`命令验证）

### 创建应用命令

```bash
bench new-app library_management
```
（将`library_management`替换为您自定义的应用名称）

### 配置应用信息

执行命令后，系统会提示您输入以下信息（可直接按Enter使用默认值）：
- **应用标题**：默认与应用名称一致
- **应用描述**：简要说明应用功能
- **发布者名称**：您的姓名或组织名
- **联系邮箱**：用于应用标识
- **应用图标**：默认使用Octicon图标（可参考octicons库）
- **主题颜色**：默认灰色
- **许可证类型**：默认MIT许可证

### 完成创建后的输出

系统将自动：
1. 在`/apps/`目录下生成应用文件夹
2. 执行依赖安装和资源构建

## 应用目录结构详解

创建完成后，应用目录结构如下：
```
apps/library_management/
├── README.md
├── library_management/          # 核心代码目录
│   ├── hooks.py                # 扩展功能的钩子文件
│   ├── modules.txt             # 模块列表定义
│   ├── patches.txt             # 数据库迁移补丁
│   ├── public/                 # 静态资源（CSS/JS）
│   ├── templates/              # Jinja模板文件
│   └── www/                    # 网页文件目录
└── pyproject.toml              # 应用构建配置
```

### 关键文件说明

- **hooks.py**：用于自定义应用行为（如安装流程、界面扩展等）
- **modules.txt**：定义应用包含的功能模块
- **pyproject.toml**：可添加Python第三方依赖

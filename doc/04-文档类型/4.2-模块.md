# 模块

在Frappe框架中，**模块（Module）** 是用于对相关数据模型（DocType）进行分组的基本单位。每个DocType都必须属于一个模块，以便更好地组织和管理。

## 主要功能与特点

### 1. 分组管理
模块帮助将功能相关的DocType归类在一起，便于查找和维护。

### 2. 目录结构
模块也对应着代码文件的目录结构。例如，一个模块的控制器文件会存放在其对应的模块目录下。

### 3. 内置模块示例
Frappe框架内置了多个标准模块，例如：

#### Core（核心模块）
包含DocType、DocField、Report、System Settings等基础模型。

#### Desk（桌面模块）
包含ToDo、Event、Note、Kanban Board等与桌面操作相关的模型。

#### Email（邮件模块）
包含Email Account、Newsletter、Email Group等邮件功能模型。

## 目录结构示例

```
frappe
├── core
│   ├── doctype
│   │   ├── doctype
│   │   ├── docfield
│   │   └── system_settings
├── desk
│   ├── doctype
│   │   ├── todo
│   │   ├── event
│   │   └── note
```

## 总结

模块是Frappe框架中用于组织DocType的重要概念，它不仅影响数据模型的分类，还直接关联到代码文件的存储结构。通过模块，开发者可以更高效地管理功能模块和代码资源。

如果您需要进一步了解特定模块的DocType列表或自定义模块的方法，可以参考Frappe文档中的"Doctypes"和"Apps"相关章节。

# 文档字段

DocField 定义了 DocType 的一个属性（或字段）。您可以定义 DocFields 的列名、标签、数据类型等。例如，ToDo doctype 有 description、status 和 priority 字段。这些最终成为数据库表 tabToDo 中的列。

## 示例

DocField 存储关于字段的元数据。其中一些描述如下：

```json
[
 {
 "label": "Description", // 向用户显示的值（表单、打印等）
 "fieldname": "description", // 我们在代码中引用的属性名，也是列名
 "fieldtype": "Text Editor", // 决定如何存储此值的字段类型
 "reqd": 1 // 此字段是否为必填项
 },
 {
 "label": "Status",
 "fieldname": "status",
 "fieldtype": "Select",
 "options": [
 "Open",
 "Pending",
 "Closed"
 ]
 },
 {
 "label": "Priority",
 "fieldname": "priority",
 "fieldtype": "Select",
 "options": [ // 选择的选项列表
 "Low",
 "Medium",
 "High"
 ],
 "default": "Low" // 要设置的默认值
 },
 {
 "label": "Completed By",
 "fieldname": "completed_by",
 "fieldtype": "Link",
 "options": "User",
 "depends_on": "eval: doc.status == 'Closed'", // 此字段显示所依赖的条件
 },
 {
 "collapsible": 1,
 "collapsible_depends_on": "eval:doc.status!='Closed'", // 确定 Section Break 字段是否可折叠
 "fieldname": "sb_details",
 "fieldtype": "Section Break",
 "label": "Details"
 },
 {
 "fieldname": "amount",
 "fieldtype": "Currency", // 货币字段
 "label": "Amount",
 "non_negative": 1, // 确定此字段值是否可以为负数
 "options": "INR",
 }
]
```

## 新属性

类似于 `depends_on` 属性（确定字段是否显示），在版本 12 中我们引入了两个新属性：

- **mandatory_depends_on**：如果满足此条件，字段将成为必填项。
- **read_only_depends_on**：如果满足此条件，字段将成为只读字段。

Frappe 开箱即用地提供了超过 30 种不同的字段类型。这些字段类型服务于各种用例。您可以在下一页中了解更多关于字段类型的信息。

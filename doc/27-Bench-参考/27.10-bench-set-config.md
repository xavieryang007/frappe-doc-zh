# 27.10 bench set-config

## **用法**
```bash
bench set-config [选项] KEY VALUE
```

## **描述**
Bench 提供了一个包装器命令来插入或更新站点配置文件中的值。您可以通过同一个命令更新站点的 `site_config.json`，以及 bench 目录的 `common_site_config.json`。

要了解更多关于站点配置和了解键值优先级的信息，请参阅[相关文档](https://docs.frappe.io/framework/user/en/bench/reference/set-config)。

## **标志**
- `-g, --global`：在 Bench 的 Common Site Config 中设置值
- `-p, --parse`：解析给定的值而不是字符串。您可以使用此选项来设置字典和列表值。在早期版本中这是 `--as-dict`。

## **示例**

1. **为给定站点启用测试**：
   ```bash
   bench --site {site} set-config allow_tests true
   ```

2. **为所有站点启用测试**：
   ```bash
   bench --site all set-config allow_tests true
   ```
   使用上述命令，每个站点的 `site_config.json` 将包含键值 "allow_tests": true。这允许在所有站点上运行测试。

   ```bash
   bench set-config -g allow_tests true
   ```
   使用上述命令，bench 的 `common_site_config.json` 将包含键值 "allow_tests": true。这将允许 bench 上的每个站点运行测试，除非它们在 `site_config.json` 中为相同的键定义了值。

3. **在站点的 frappe.conf 中设置字典值**：
   ```bash
   bench --site {site} set-config backup '{"includes": ["Not", "ToDo"]}' --parse
   ```
   现在您可以在代码中访问该列表：`frappe.conf.backup.get("includes")`。

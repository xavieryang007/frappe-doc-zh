# 27.3 bench drop-site

## **用法**
`bench drop-site [选项] <站点名称>`

## **描述**
删除一个已存在的站点。此操作会删除数据库，并将该站点的文件夹从 `./sites` 移动到 `./archived_sites`（除非另行指定）。在执行删除前会先进行完整的站点备份。

## **选项**
- `--db-root-username`：指定具有删除数据库权限的数据库管理用户名称，默认为 `root`。
- `--db-root-password`：指定数据库管理用户的密码。
- `--archived-sites-path`：指定将站点文件夹移动到的路径（替代默认的 `./archived_sites`）。

## **标志**
- `--no-backup`：在删除站点前跳过备份。
- `--force`：即使遇到错误也强制删除站点。

## **示例**

1. **通过传递 root 密码跳过交互式提示**：
   ```bash
   bench drop-site <站点名称> --db-root-password <数据库 root 密码>
   ```

2. **在删除站点前跳过备份**：
   ```bash
   bench drop-site <站点名称> --no-backup
   ```

3. **将站点的文件夹移动到自定义路径而非默认的 `./archived_sites`**：
   ```bash
   bench drop-site <站点名称> --archived-sites-path <自定义存档路径>
   ```

# 27.8 bench uninstall-app

## **用法**
```bash
bench uninstall-app [选项] <应用名称>
```

## **描述**
从站点中移除应用及其链接的 DocType、模块。执行基础命令时，会在尝试删除其模块和 DocType 之前检查该应用是否存在于站点上。**运行 uninstall-app 命令时，应用不一定要安装在 Bench 环境中。**

## **选项/标志**
- `-y`, `--yes`: 绕过卸载应用的确认提示。
- `--dry-run`: 列出所有将被删除的 DocType（模拟运行，不实际执行）。
- `--no-backup`: 卸载前不备份站点。
- `--force`: 强制从站点中移除应用。

## **示例**

1. **执行模拟运行（干跑）**，查看在特定站点上运行该命令会产生什么影响：
   ```bash
   bench --site {站点名} uninstall-app {应用名} --dry-run
   ```

2. **在应用卸载操作前不进行备份**：
   ```bash
   bench --site {站点名} uninstall-app {应用名} --no-backup
   ```

3. **使用强制力从站点卸载应用**：
   ```bash
   bench --site {站点名} uninstall-app {应用名} --force
   ```

4. **跳过卸载确认的交互式提示**：
   ```bash
   bench --site {站点名} uninstall-app {应用名} --yes
   ```

## **核心要点说明**
- **主要功能**：此命令用于从一个 Frappe 站点中完全移除一个应用，包括其相关的数据表（DocTypes）和模块。
- **前置条件**：命令主要针对站点进行操作（通过 `--site` 参数指定），应用本身可以不在 Bench 的安装应用列表中。
- **风险提示**：该操作会删除数据。使用 `--dry-run` 预览将要删除的内容，并使用 `--no-backup` 选项时需明确知晓风险（除非你确信有其它备份）。
- **强制操作**：`--force` 标志用于在某些检查未通过时仍强制卸载。

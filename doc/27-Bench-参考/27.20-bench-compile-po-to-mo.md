# 27.20 bench compile-po-to-mo

## **用法**
```bash
bench compile-po-to-mo [选项]
```

## **前提条件**
您的应用程序的 `locale` 目录中需存在至少包含一种语言翻译的 PO 文件。

## **描述**
PO 文件以人类可读的格式保存翻译。在应用程序使用这些翻译之前，需要将它们编译成二进制文件（扩展名为 MO）。生成的二进制文件将保存在您 bench 的 `sites/assets/locale` 文件夹中。

为了提高效率，如果已存在的 MO 文件比 PO 文件更新，此命令将跳过编译过程。

此命令会在执行 `bench build` 和 `bench update` 时自动运行。在开发环境中，当您刚添加了一些翻译并希望立即在用户界面中查看结果时，可以显式地运行此命令。

## **选项**
- `--app`：此参数允许您指定一个应用程序，以便仅更新该应用的 PO 文件。默认值：所有应用。
- `--locale`：此参数允许您指定一个语言环境，以便仅更新该语言环境的 PO 文件。默认值：所有语言环境。
- `--force`：强制重新编译，即使 PO 文件没有更改。

## **示例**
**编译 ERPNext 的德语翻译**：
```bash
bench compile-po-to-mo --app erpnext --locale de
```

此命令会从 `apps/erpnext/erpnext/locale/de.po` 读取翻译并将其编译为二进制格式。结果将保存到 `sites/assets/locale/de/LC_MESSAGES/erpnext.mo`。

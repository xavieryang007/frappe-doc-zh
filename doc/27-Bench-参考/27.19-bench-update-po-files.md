# 27.19 bench update-po-files

## **用法**
```bash
bench update-po-files [选项]
```

## **先决条件**
- 您的应用（app）的 `locale` 目录中必须存在 POT 文件和 PO 文件。

## **描述**
此命令会根据现有的 POT 文件（包含所有可翻译字符串）同步您现有的翻译文件（PO 文件）。它会移除所有过时的字符串并添加新的字符串。

在您使用 `bench generate-pot-file` 等命令重新生成了包含可翻译字符串的 POT 文件之后，您需要运行此命令。如果您使用专门的翻译管理平台，该平台可能也会为您处理此步骤，那么您就不需要手动运行此命令。

## **选项**

| 选项 | 描述 |
| :--- | :--- |
| `--app` | 此参数允许您指定一个应用，以便仅更新该应用的 PO 文件。**默认值**：所有应用。 |
| `--locale` | 此参数允许您指定一个地区（语言代码），以便仅更新该地区的 PO 文件。**默认值**：所有地区。 |

## **示例**

**同步 ERPNext 的可翻译字符串与存储德语翻译的文件**：
```bash
bench update-po-files --app erpnext --locale de
```

此命令会从 `apps/erpnext/erpnext/locale/de.po` 读取翻译，并从 `apps/erpnext/erpnext/locale/main.pot` 读取可翻译字符串。结果将保存回 `apps/erpnext/erpnext/locale/de.po`。

## **总结**
该命令主要用于在 POT 模板文件更新后，同步并更新具体的语言翻译文件（.po），确保翻译文件包含最新的待翻译字符串，并清理已不存在的字符串。
